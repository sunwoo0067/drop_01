## Blocker(필수 수정)
1) DB 트랜잭션 안에서 외부 API 호출
- `get_session()`이 `with session.begin():` 트랜잭션 범위를 endpoint 전체로 잡고 있습니다.
- `POST /api/coupang/register/{product_id}` → `register_product()` 호출 과정에서 쿠팡 HTTP 호출이 트랜잭션 안에서 수행되어 장시간 트랜잭션/락/대기/동시성 저하 위험이 큽니다.
- 권장: 쿠팡 API 호출은 트랜잭션 밖으로 분리하거나, `BackgroundTasks`(또는 큐)로 넘기고 `202 Accepted`로 응답 후 상태를 업데이트하는 방식으로 변경해주세요.

2) `BackgroundTasks` 파라미터 미사용
- `app/api/endpoints/coupang.py`에서 `background_tasks`를 받고 있지만 실제 사용하지 않습니다.
- 동기 처리면 파라미터 제거, 비동기 처리면 실제로 `background_tasks.add_task(...)`를 사용하도록 정리 부탁드립니다.

## High(중요 수정)
3) `CoupangClient.update_price()` 쿼리 파라미터 미반영 버그
- `update_price()`에서 `params`를 만들었지만 `self.put(..., payload=None)`로 호출되어 `forceSalePriceUpdate`가 URL query string에 붙지 않습니다.
- 이미 `_update_price_internal()`에 올바른 구현이 있으니 `update_price()`가 이를 사용하도록 수정하는 편이 안전합니다.

## 권장 개선(Nice to have)
4) API 에러 메시지/로그 한국어 통일

5) `/api/products` 응답 모델 지정
- 현재 ORM 객체를 그대로 반환합니다. 응답 스키마(Pydantic response model)를 명시해서 반환 필드를 제한하면 안정적입니다.
