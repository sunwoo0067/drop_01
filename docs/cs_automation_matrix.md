# CS Automation Matrix (v1.7.1)

본 문서는 AI CS 에이전트의 자동 답변 송출(AI_DRAFTED) 여부를 결정하는 다중 벡터 기준을 정의합니다.

## 1. 개요 (Overview)
단순 신뢰도 점수(Confidence Score)만으로는 복잡한 비즈니스 리스크를 관리할 수 없습니다. 따라서 문의의 '의도(Intent)', 제품의 '카테고리(Category)', 그리고 '신뢰도'를 종합하여 자동화 여부를 결정합니다.

## 2. 벡터별 리스크 정의

### A. 문의 의도 (Intent)
| 리스크 단계 | 주요 의도 | 자동화 방침 |
| :--- | :--- | :--- |
| **LOW** | 배송조회, 배송기간, 단순 사용법, 재고확인 | 적극 권장 (신뢰도 중심) |
| **MEDIUM** | 취소문의, 옵션변경, 상품 상세정보 확인 | 조건부 허용 (카운터 체크 필요) |
| **HIGH** | 반품/교환 요청, 파손/오배송 클레임, 법적 대응 | **수동 검토 필수** (Human-in-the-Loop) |

### B. 상품 카테고리 (Category)
| 리스크 단계 | 상품군 예시 | 자동화 가산점 |
| :--- | :--- | :--- |
| **SAFE** | 생활잡화, 사무용품, 인테리어 소품 | +0.0 (기본값) |
| **CAUTION** | 일반가전, 아웃도어 의류, 완구 | -0.1 (임계값 상향) |
| **CRITICAL** | 의료기기, 대형가전, 식품, 고가 명품 | -0.3 (사실상 수동 전환) |

## 3. 최종 판정 로직 (Decision Logic)

최종 판정은 다음 수식에 따라 결정됩니다:
`Final_Score = Confidence_Score - Intent_Risk_Weight - Category_Risk_Weight`

| Final_Score 범위 | 최종 상태 | 조치 |
| :--- | :--- | :--- |
| **>= 0.85** | `AI_DRAFTED` | 자동 승인 및 대기 |
| **< 0.85** | `HUMAN_REVIEW` | 운영자 확인 필요 |

> [!IMPORTANT]
> **반드시 수동 검토가 필요한 예외 상황:**
> - 문의 내용에 'Legal', '소송', '변호사', '공정위' 등 키워드 포함 시
> - 감정 분석 결과 'Strong Anger' (매우 화남) 감지 시
> - 고객이 이전에 2회 이상 동일 건으로 재문의했을 시
